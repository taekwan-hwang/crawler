기존의 코드를 보면 CLS: 이후부분에서 화면을 clear 해주고 있습니다.
과제 1번을 하기위해서 clear한 다음에 문자열을 출력하거나( CLS: ~ 블록 다음에 코딩)  START: 부분에서 offset이 맞춰져 있으므로 [es:bx] 부분에 직접주소지정방식으로 출력 하고 다음에 CLS: 블록을 수행하면 본인의 문자열 다음을 clear 해주게 됩니다 ~

넘어간다는게 처음 화면이 뜬후 CLS분을 통해 화면이 clear되고 Hello world가 뜬다는 의미였습니다.
cls 블록 다음에 print블록을 만들고 start 에서 call print를 했는데 이미지처럼 Booting from Hard Disk...가 뜨고 멈춰있네요.
저 오류는 왜 뜨는지 알 수 있을까요?

혹시 jmp $  <- 이 명령어가 들어가 있는지 확인해보세요 무한루프를 돌려주는 구문으로 위치에 따라서   booting from hard disk에서 안넘어가고 계속 루프를 돌 수 있습니다.

무한루프가 CLS에서 계속 도는건가요? 그럼 jmp $명령어가 CLS안에 들어가야하나요?

start 부분과 cls 부분은 윗분이 설명해주신것 처럼 화면을 clear해주기 위해서 사용합니다.  밑에 분이 설명해 주신것 처럼 
cx(카운터 레지스터)값에 화면 크기인 가로 80*세로 25* 글자크기 *2 만큼의 횟수의 루프를 돌면서 비디오메모리영역을 0으로 채워서 화면을 clear 해주는 방식으로 알고 있습니다.(조금 불확실) 
loop 1회당 cx값이 1줄어드는 형식으로요.  인터넷에서 검색해 보실 때 몇몇 Hello world 출력 코드에 jmp $ 가 들어가 있는 경우가 있습니다.  
jmp $ 에서 $는 현재 주소를 의미하며 현재주소로 점프하라는 명령은 사실상 무한 루프인 것이죠. 그래서 만약에 이 코드가 CLS 전에  들어가 있으면 booting from hard disk 에서 CLS  로 안넘어가서 멈춰 있는경우가 있어서 말씀드린겁니다. 
현재 작성하신 코드에서 jmp $ 명령어가 들어있을경우 질문현상이 일어날 수 있어서 체크해 보라는 말이었습니다.

CLS 구문은 cx register로 loop횟수를 정해주고 사용하고 있는걸로 알고있습니다.           
jmp $ 현재 명령어로 분기하라는 뜻으로 c언어의 while(1)과 비슷한 의미인걸로 알고있습니다.  cls안에 들어가는거는 조금 이상한거 같아요 !
