(사진 첨부 )
정확히 ata.c의 ata_read함수에서 멈춥니다.

멈추는 위치는 계속 다릅니다.

(추가 질문)
계속 막혀서 gdb를 쓸라고 하는데 gdb는 아예 진행이 안됩니다. 이번 과제에서 GDB 사용 안돼나요?



==============Comment================
해당 질문은 소스코드를 직접 보지 않으면 정확한 판단이 어려운 부분입니다. 하지만 비슷한 문제를 겪은 다른 학우들의 경우 fs_read/write에서 startsec 값을 blcok index가 아닌 disk의 물리적인 주소로 생각해서 비슷한 문제가 발생했었습니다. 비슷한 실수를 하셨다면 아래 다른 질문들 답변을 참고하시기 바랍니다.

ata.c의 ata_read 함수에서 멈추는데 ata_read도 수정해냐 하나요?

아닙니다. ata함수는 수정하실 필요가 없습니다. ata_Read에서 멈춘다는 것은 DEVOP_READ/WRITE 두번째 인자로 전달되는 블록인덱스가 잘못되었다는 것입니다.

이번 과제에서 주로 문제가 되는것은 대부분 disk 관련 문제이기 때문에 gdb사용은 거의 의미가 없습니다.
