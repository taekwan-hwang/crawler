==============Content================
active array 는 기존의 스케줄러에서 사용하던 fifo 형태의 slist , rlist 등을 대체하는 것으로서, list  2차원배열로 각각의 배열의방은 priority 에 맞는 run / sleep 의 2가지 status를 갖는 process element를 갖는 스케줄링의 기준점이라고 이해했습니다. 
재대로 이해했는지 모르겠습니다만, 만일 기존의 fifo를 대체하게 된다면 process.c 에 있는 기존 fifo list 를 인자로 받는 많은 함수들도 수정이 필요하게 되는것은 아닌가하는 부분에 있어 제가 이해한바를 조금 의심하고있는 상황입니다..
제가 이해한바가 맞는것인지요? 맞다면 기존 인자를 받던 함수의 수정이 대공사라 할만큼 필요하게될지( 설마 ) 그것도 궁금합니다. 많은 일침 부탁드립니다.



==============Comment================
1. run list만 2차원 배열로 구현하시면 됩니다. sleep의 list는 통틀어서 하나로만 관리하도록 구현하시면 됩니다.
2. 기존 fifo 스케줄러를 대체하기 위해서 schedule()함수는 대거 수정이 필요할 것이고, 그 외에 schedule 관련된 함수들도 약간의 수정이 필요할 것입니다.

명세상에 active 배열에 I/O인터럽트( proc_sleep() 대체)로 status 가 변한(PROC_STOP) 프로세스들도 이 배열에 존재한다고 나와있는것 같은데요 이게 active가 run/ sleep 둘다 원소로 갖는다는 말이 아닌것인가요?

기존 소스코드를 분석해보시면 눈치채셨겠지만 status가 변한 프로세스는 다른 리스트로 옮겨갑니다. I/O 인터럽트가 발생하는 상황으로 인해서 time slice를 전부 사용하지 못한 프로세스만 active array에 존재하면 된다는 뜻이므로 active에는 run, sleep 모두를 가져야 할 필요는 없습니다.
